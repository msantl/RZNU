Resurs Metode
/users/
Tip Ostvarenje
Create POST - stvara novog korisnika ako se u zahtjevu nalazi login podaci administratora. U zahtjevu je potrebno poslati username i password.
Read GET - vraca JSON objekt koji sadrzi id i username svih korisnika
Update PUT - korisniku ciji id je zadan promijeni usernem i/ili password. Potrebno je zadati id korisnika.
Delete DELETE - ako su u zahtjevu podaci administratora, obrisu se svi ostali korisnici osim administratora zajedno s njihovim snippetima
/snippets/
Create POST - stvara se novi snippet pod korisnikom ciji se podaci nalaze u zahtjevu. Potrebno je zadati naslov, jezik te izvorni tekst snippeta kao i login podatke korisnika
Read GET - dohvaca se JSON objekt koji sadrzi sve snippete.
Update PUT - snippetu ciji je id zadan promijene se naslov i/ili jezk i/ili izvorni tekst. Potrebno je da zahtjev sadrzi login podatke korisnika ciji se snippet mijenja
Delete DELETE - brisu se svi snippeti iz sustava. Potrebno je navesti login podatke administratora.
/users/X
Create POST - metoda nije dozvoljena
Read GET - dohvaca se JSON objekt koji sadrzi sve snippeta korisnika ciji je id zadan u pathu.
Update PUT - mijenja se username i password korisniku ciji je id naveden u pathu. Potrebno je da je taj id jednak korisniku ciji se login podaci nalaze u zahtjevu.
Delete DELETE - brisu se podaci o korisniku te svi njegovi snippeti. Potrebno je navesti podatke korisnika kojeg zelimo obrisati iz sustava u zahtjevu.

Create POST - metoda nije dozvoljena

Update PUT - snippetu ciji je id zadan u pathu promijene se naslov i/ili jezk i/ili izvorni tekst. Potrebno je da zahtjev sadrzi login podatke korisnika ciji se snippet mijenja
Delete DELETE - snippet ciji se id nalazi u pathu se brise iz sustava
